## 🛠️ New tools gaining traction

- **Cursor Composer 2.5** — Cursor's first in-house frontier coding model, built on a Kimi K2.5 base with 85% of compute spent on Cursor's own RL post-training.
  *Traction:* matches Claude Opus 4.7 and GPT-5.5 on SWE-Bench Multilingual (79.8%) and Terminal-Bench 2.0 (69.3%) at one-tenth the per-token cost ($0.50 input / $2.50 output vs Opus 4.7's $5 / $25); Cursor shipped double-usage promo for launch week.
  *Why it matters:* the cheapest path to frontier-grade coding agents is now bundled inside an IDE that already has the developer mindshare. [source](https://the-decoder.com/cursors-composer-2-5-matches-opus-4-7-and-gpt-5-5-benchmarks-at-a-fraction-of-the-cost/)

  *Traction:* launched May 19 at Google I/O alongside Gemini Spark and Omni; Gemini 3.5 Flash runs at 289 output tokens/sec — roughly 4x faster than Opus 4.7 or GPT-5.5 at comparable benchmarks.
  *Why it matters:* fan-out agent workloads (research, monitoring, batch automation) are now economically viable on Google's stack at a speed advantage that compounds in long-running pipelines. [source](https://www.axios.com/2026/05/21/google-ai-anthropic-openai-war)
